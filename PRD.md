# 2026世界杯300-Agent预测系统 PRD

## 1. 项目背景

世界杯是全球最大体育IP，预测类内容天然有高传播性。但单一专家视角容易陷入"押强队"或"押黑马"的二元对立。

本项目尝试用**群体智慧 + 派别分歧**的方法：
- 300个独立AI agent分别预测
- 强制分成10个不同"世界观"派别（数据派/赔率派/玄学派等），保证判断**真正分歧**而非全员押同一支队
- 加权聚合输出**派别热力榜 + 分歧图**，呈现"派别想法"而非简单平均

**仅供娱乐**，不构成任何投注建议。

## 2. 核心目标

| 目标 | 验收标准 |
|------|---------|
| 派别真实分歧 | 每个派别的Top1冠军不能全部相同（必须≥5个不同冠军） |
| 300份真实独立判断 | 每个agent都是M3大模型真实生成的结果，禁止模板/随机 |
| 真实数据基础 | agent判断时联网检索最新FIFA排名、赔率、伤病、赛程 |
| 可视化交付 | 单文件HTML可部署，三区交互（过程/结果/分派别） |
| 用户参与 | 进入结果区前用户先押冠军，最后高亮"你和哪个派别一致" |

## 3. 派别设计（10派 × 30人 = 300人）

每个派别是一套"世界观"，决定agent如何加权。

| 派别 | 加权逻辑 | 预期输出倾向 |
|------|---------|------------|
| 数据派 (data) | FIFA积分、ELO、xG、阵容身价 | 纸面最强（法/英/巴） |
| 赔率派 (odds) | 博彩盘口隐含概率 | 庄家最爱 |
| 老球迷派 (history) | 历史底蕴、决赛抗压、夺冠基因 | 传统豪门 |
| 玄学派 (mystic) | 东道主魔咒、卫冕魔咒、偶数年规律 | 反主流找冷门 |
| 主帅视角派 (coach) | 战术体系克制、排兵布阵深度 | 看临场调度 |
| 伤病赛程派 (injury) | 关键球员伤病、赛程难度、气候适应 | 拼"完整度" |
| 黑马派 (darkhorse) | 专挑被市场低估、有冷门潜力 | 高反差输出 |
| 阵容年龄派 (age) | 核心当打之年 vs 青黄不接 | 看"窗口期" |
| 心理抗压派 (mental) | 点球记录、逆境翻盘、淘汰赛大心脏 | 拼"心脏" |
| 建模派 (model) | 蒙特卡洛/泊松模拟 | 量化最优 |

**派别内差异化**：每个agent有不同身份（不同国家老球迷、不同联赛主帅、不同信仰体系的老球迷），确保理由不雷同。

## 4. Agent协议

每条输出严格结构化：

```json
{
  "id": "V01",
  "faction": "mystic",
  "identity": "1998年法国世界杯玄学派老球迷",
  "champion": "法国",
  "top3": ["法国", "巴西", "西班牙"],
  "confidence": 75,
  "reason": "卫冕冠军+东道主身份+姆巴佩巅峰期",
  "weight_logic": "偶数年+卫冕+玄学三件套"
}
```

字段约束：
- `champion`: 1支队，从2026世界杯48支参赛队中选
- `top3`: 3支队，可重复包含champion
- `confidence`: 0-100整数，≥80是高信心
- `reason`: ≤80字，不许用模板套话

## 5. 聚合算法

### 5.1 加权投票
```
vote_weight(agent, team) = confidence/100 × faction_weight × bias_factor
```

faction_weight（派别整体加权）：

| 派别 | 权重 | 理由 |
|------|-----|------|
| 数据派 | 1.3 | 数据扎实 |
| 赔率派 | 1.5 | 市场反映所有信息 |
| 建模派 | 1.4 | 量化最稳 |
| 玄学派 | 0.8 | 娱乐向 |
| 黑马派 | 0.6 | 反向指标 |

### 5.2 概率计算
```
team_probability = Σ vote_weight(agent, team) for all agents who voted team as champion
```

然后归一化到100%。

### 5.3 热度榜
取Top8进入热力榜，按probability降序。

### 5.4 黑马识别
```
gap(team) = dark_faction_pct(team) - odds_faction_pct(team)
divergence(team) = std(all 10 factions' pct for team)
```

最被低估黑马 = max(gap × divergence)的队。

### 5.5 分派别分歧
输出"派别A选X、派别B选Y"的对比矩阵，直观体现判断分歧。

## 6. 网页交付

### 6.1 三区结构
1. **过程区**（运行中显示）：300个agent头像网格，逐个亮起 + 各派别代表性判断滚动
2. **结果区**（完成后显示）：Top8热力榜 + count-up入场 + 最大热门/最被低估黑马一句话点评
3. **分派别图区**（结果后显示）：按派别切换，看哪些agent押谁，柱状对比

### 6.2 视觉规范
- 底色：`#FBF9F1`（米白纸感）
- 文字：`#13233A`（深藏青墨）
- 强调：`#9B2D20`（砖红，关键数字）或 `#C8A45C`（冠军金）
- 字体：标题紧凑大写无衬线，数据等宽
- 动效：count-up、条形入场、hover微交互

### 6.3 交互钩子
- **押注环节**：进入结果区前让用户选一支自己心目中的冠军
- **反馈高亮**：
  - "你和【XX派】想法一致"
  - "比主流共识更保守/更大胆"
  - "X / 300个agent和你同选"

### 6.4 技术约束
- 单文件HTML，所有CSS/JS/数据内嵌
- 无外部依赖（除CDN字体）
- 可部署到任何静态服务器（已上线 minimax space）

## 7. 反造假约束

- ❌ 不允许循环/模板/随机文案伪造视角
- ❌ 不允许先写好N句话再随机分配
- ❌ 不允许agent输出"无判断"或"弃权"
- ✅ 每个agent必须是M3大模型真实生成的JSON
- ✅ 理由字段必须真实反映agent的人设和加权逻辑
